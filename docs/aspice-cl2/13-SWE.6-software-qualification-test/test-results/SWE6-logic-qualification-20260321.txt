===============================================================================
DEFENSE RATIONALE — WHY LOGIC TESTING ON SIL IS VALID
===============================================================================

1. SAME SOURCE CODE: The foxBMS C source compiled for x86 is identical to
   the TMS570 target. Only HAL stubs differ (hardware abstraction).

2. SAME DIAG THRESHOLDS: diag_cfg.c threshold values (50, 500, 10 events)
   are compiled identically. The counting logic is architecture-independent.

3. SAME STATE MACHINE: BMS state transitions follow the same code path
   regardless of instruction set (ARM vs x86).

4. LOGIC ≠ TIMING: "BMS SHALL open contactors on OV" is a logic requirement.
   "BMS SHALL open contactors WITHIN 250ms" is a timing requirement.
   We verify the former. The latter needs HIL.

5. ISO 26262 Part 6 Clause 9 permits "back-to-back" testing where the
   same test is run on model/SIL and target. Logic results transfer;
   timing results do not.

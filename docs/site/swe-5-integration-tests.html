1. Purpose

This document specifies integration-level tests for the foxBMS 2 POSIX port, satisfying ASPICE SWE.5 (software integration and integration test) and ISO 26262 Part 6 Clause 10. Integration tests verify that modules interact correctly through their defined interfaces.

2. Scope

Integration tests verify: - CAN interface data flow (external stimulus to internal database) - Database producer/consumer data integrity across modules - State machine transitions driven by inter-module interactions - Safety path: SOA to DIAG to BMS to Contactor

  1. Level 1: HAL stubs + Driver layer (CAN via SocketCAN, SPI/I2C stubs)
  2. Level 2: Engine layer (Database, DIAG) on top of drivers
  3. Level 3: Application layer (SOA, BMS, Algorithm) on top of engine
  4. Level 4: Full system integration with CAN stimulus
